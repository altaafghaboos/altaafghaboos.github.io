PyCryptoDome Audit

v-hunting

v-patching

On the first day of Project Stormwing, only 3 hours after launch, we caught our first vulnerability.

PyCryptoDome is a common python library that's used in over 8,000 open-source software around the world, and the vulnerability we caught had been lying there for 8 years.

Basically, there was a mistake in the parameters. The library uses Ed25519, and from NIST SP 800-38D, plaintext input data cannot be more than 2^39 - 256 bits. However, in the parameters, user input was indicated to be in bytes, and the check for the size of the input was checking that the input was not to be more than 2^39 - 256 bytes.

We flagged it as CWE-1240 (use of cryptographic primitive with a risky implementation), and patched it simply by adding "//8" to the check (division operation done on the number).

Again, officially it wasn't me alone who found it as Logan was helping us through it, but still, for this one my name is on the documentation ;)
